# Pal-Plant Implementation Plan: Addressing Gaps & Challenges

This implementation plan outlines the technical requirements and execution steps for resolving the primary gaps identified in the Pal-Plant application, specifically focusing on mobile readiness, accessibility, and native feature integration.

---

## 1. Mobile Readiness: Bridging the Capacitor Gap
This phase focuses on ensuring native mobile features function reliably on Android and iOS platforms.

### **Native Permission Infrastructure (Issue #22)**
* **Centralized Manager**: Create a `PermissionsManager` utility in `pal-plant/utils/` to handle status checks and request logic for `notifications` and `contacts` using `@capacitor/core`.
* **Permission Primers**: Implement educational modals in `components/` that explain the benefit of enabling notifications (e.g., "Don't let your plants wilt") before triggering system prompts.

### **Mobile Push Debugging (Issue #21)**
* **Build Targeting**: Configure the build process to ignore `firebase-messaging-sw.js` for native builds, as Capacitor handles push through native SDKs rather than Service Workers.
* **Token Diagnostic**: Add a "System Status" section in the `SettingsModal` to verify if a valid FCM token has been generated by the device.

### **Mobile Parity Audit (Issue #24)**
* **Hit Targets**: Audit all interactive elements in `FriendCard.tsx` and `MeetingRequestsView.tsx` to ensure touch targets meet the minimum 44x44 pixel standard.
* **Platform-Specific CSS**: Use Tailwind's platform-specific modifiers or media queries to adjust layouts for smaller mobile screens.

---

## 2. UI/UX Refinement: Theme & Hierarchy
This phase improves visual accessibility and the efficiency of the relationship management interface.

### **Theme Contrast Fixes (Issue #25)**
* **Semantic Tokens**: Replace hardcoded Tailwind colors in components with CSS variables defined in `index.css` to allow for global contrast adjustments.
* **Midnight Theme Recalibration**: Update the color palette for the "Midnight" theme to ensure a minimum 4.5:1 contrast ratio (WCAG AA) for all plant status indicators.

### **Garden Entry Redesign (Issue #28)**
* **Information Priority**: Redesign the `FriendCard` to visually prioritize the "Interaction Weight" buttons (Text, Call, In-Person) and the "Last Contacted" status.
* **Nudge Integration**: Display smart suggestions from `utils/nudges.ts` directly on the card to guide the user on the best way to "water" a specific plant.

---

## 3. Contact Management: Native Integration & Flexibility
This phase expands how users define and import their social data.

### **Native Contact Import (Issue #23)**
* **API Integration**: Implement `@capacitor-community/contacts` to allow direct browsing and importing from the device's native contact list.
* **Data Mapping**: Update the import logic to map native fields to the `Friend` interface, specifically populating the `phone` and `email` fields.

### **Meeting Timeframe Selector (Issue #26)**
* **Schema Extension**: Add an optional `desiredTimeframe` (e.g., Morning, Afternoon, Evening) to the `MeetingRequest` interface in `types.ts`.
* **Scoring Update**: Modify `useFriendsEngine.ts` to award bonus points to the "Social Garden Score" when meetings are completed within the user’s preferred window.
